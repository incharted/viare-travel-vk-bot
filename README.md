# VIARE Travel VK Bot

VIARE Travel VK Bot - чат-бот для сообщества ВКонтакте, который помогает туристическому проекту консультировать клиентов, подбирать туры, принимать заявки и обрабатывать обращения через панель менеджера и администратора.

Проект сделан как учебная разработка по теме «Разработка чат-бота для консультации клиентов». Внутри бота объединены клиентский сценарий, каталог туров, хранение заявок в базе данных, роли сотрудников, экспорт данных и простая CRM-логика прямо в сообщениях ВКонтакте.

## Возможности

- Подбор тура по шагам: формат поездки, направление, даты, количество туристов, бюджет и тип отдыха.
- Каталог туров по России и зарубежным направлениям.
- Карточки туров с описанием, длительностью, ценой и расчетом стоимости.
- Личный кабинет клиента: мои заявки, мой статус, избранные туры.
- Избранное и сравнение туров.
- Создание заявки после подтверждения данных клиентом.
- Уведомления менеджеру о новой заявке.
- Панель менеджера для просмотра, закрепления, обработки и закрытия заявок.
- Статусы заявок: новая, в работе, ждет клиента, закрыта.
- Панель администратора для управления менеджерами, турами, пользователями и контентом.
- Экспорт заявок в CSV и XLSX.
- Генерация PDF-коммерческого предложения.
- FAQ, контакты, условия оплаты и бронирования.
- Базовая защита от спама и слишком частых сообщений.
- Поддержка локальной SQLite-базы данных.

## Структура проекта

```text
.
|-- main.py                    # точка входа приложения
|-- config.py                  # загрузка настроек из переменных окружения
|-- database/                  # подключение SQLite и схема базы данных
|-- handlers/                  # обработчики сообщений VK
|-- keyboards/                 # кнопки и клавиатуры VK
|-- middlewares/               # аудит, антиспам, нормализация текста, rate limit
|-- services/                  # бизнес-логика проекта
|-- utils/                     # валидаторы, форматирование, состояния, логирование
|-- tests/                     # автоматические тесты
|-- scripts/                   # PowerShell-скрипты для Windows
|-- setup_bot.bat              # установка окружения
|-- run_bot.bat                # запуск бота
|-- cleanup_bot.bat            # очистка временных файлов
|-- requirements.txt           # зависимости Python
|-- Dockerfile                 # запуск в контейнере
|-- .env.example               # пример файла настроек
```

## Быстрый запуск на Windows

Установите Python 3.11 или новее, затем откройте PowerShell в папке проекта и выполните:

```powershell
.\setup_bot.bat
```

Скрипт создаст виртуальное окружение, установит зависимости и подготовит файл `.env`, если его еще нет.

Если PowerShell блокирует выполнение скриптов, выполните команду:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

После этого снова запустите установку:

```powershell
.\setup_bot.bat
```

## Настройка

В корне проекта должен быть файл `.env`. Его можно создать из примера:

```powershell
copy .env.example .env
```

Минимально нужно заполнить:

```env
VK_TOKEN=your_token_here
ADMIN_IDS=123456789
```

Основные переменные:

| Переменная | Назначение |
| --- | --- |
| `VK_TOKEN` | Токен сообщества ВКонтакте с доступом к сообщениям. |
| `ADMIN_IDS` | VK ID администраторов через запятую. |
| `MANAGER_IDS` | VK ID менеджеров через запятую. |
| `DB_PATH` | Путь к SQLite-базе данных. |
| `LOG_FILE` | Путь к файлу логов. |
| `RATE_LIMIT_COUNT` | Количество сообщений за период антиспама. |
| `RATE_LIMIT_WINDOW_SEC` | Длина периода антиспама в секундах. |
| `EXTERNAL_TOURS_ENABLED` | Включение внешнего поиска туров. |

Файл `.env` не предназначен для публикации, потому что содержит токен и локальные настройки. Для примера настроек используется только `.env.example`.

## Роли в боте

### Клиент

Клиент работает с главным меню. Он может подобрать тур, посмотреть каталог, открыть избранное, проверить свои заявки, прочитать FAQ, контакты и условия оплаты, а также связаться с менеджером.

### Менеджер

Менеджер получает доступ к рабочей панели командой:

```text
/staff
```

В панели менеджера доступны заявки, карточки заявок, закрепление обращения за собой, ответ клиенту, шаблоны ответов, закрытие заявок и просмотр рабочей нагрузки.

### Администратор

Администратор открывает панель командой:

```text
/admin
```

Администратор может управлять менеджерами, турами, пользователями, текстовыми блоками, экспортом заявок, PDF-коммерческими предложениями и рассылками. Администратор также автоматически имеет права менеджера.

## Тестирование

В проекте есть автоматические тесты для проверки валидаторов, фильтрации туров, антифрода и ограничения частоты сообщений.

Запуск тестов:

```powershell
python -m pytest -q
```

## Очистка локальных данных

Для удаления временных файлов, логов и runtime-данных можно использовать:

```powershell
.\cleanup_bot.bat
```

Локальные рабочие файлы создаются в папке `data/`. Они не публикуются в репозитории, потому что содержат базу данных, логи, lock-файлы и выгрузки.
