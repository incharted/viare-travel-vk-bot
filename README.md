# VIARE Travel VK Bot

VIARE Travel VK Bot - чат-бот для сообщества ВКонтакте, который помогает консультировать клиентов туристического проекта, подбирать туры, оформлять заявки и обрабатывать обращения через менеджерскую и административную панели.

Проект объединяет клиентскую часть и простой CRM-процесс внутри сообщений ВКонтакте: клиент выбирает направление, указывает даты, бюджет и количество туристов, подтверждает заявку, после чего менеджер получает обращение и может ответить клиенту прямо из бота.

## Возможности

- Чат-бот для сообщений сообщества ВКонтакте на базе `vkbottle`.
- Пошаговый подбор тура: Россия или зарубеж, направление, даты, количество туристов, бюджет, тип отдыха и подтверждение.
- Каталог туров с карточками, избранным и сравнением вариантов.
- Личный кабинет клиента: заявки, текущий статус, избранные туры.
- Статусы заявок: новая, в работе, ждет клиента, закрыта.
- Панель менеджера для обработки заявок, ответов, шаблонов и просмотра загрузки.
- Панель администратора для управления менеджерами, турами, пользователями, контентом, рассылками, экспортом и PDF КП.
- База данных SQLite для пользователей, заявок, туров, логов, текстовых блоков и избранного.
- Экспорт заявок в CSV и XLSX.
- Формирование PDF-коммерческих предложений.
- Базовая защита от спама и слишком частых сообщений.
- Скрипты для установки, запуска и очистки проекта на Windows.
- Настройки для подключения внешнего источника туристических предложений.

## Установка

### Windows

Установите Python 3.11 или новее, затем выполните:

```powershell
.\setup_bot.bat
```

Скрипт создаст виртуальное окружение, установит зависимости и создаст `.env` из `.env.example`, если файла `.env` еще нет.

Если PowerShell блокирует выполнение скриптов, откройте PowerShell в папке проекта и выполните:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

Эта команда действует только для текущего окна PowerShell. После нее можно снова запустить:

```powershell
.\setup_bot.bat
```

### Ручная установка

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
```

## Настройка

Создайте файл `.env` в корне проекта и заполните основные переменные.

```env
APP_ENV=dev
VK_TOKEN=your_token_here
ADMIN_IDS=
MANAGER_IDS=
DB_PATH=./data/bot.sqlite3
LOG_LEVEL=INFO
LOG_FILE=./data/bot.log
RATE_LIMIT_COUNT=8
RATE_LIMIT_WINDOW_SEC=10
VK_SSL_VERIFY=1
EXTERNAL_TOURS_ENABLED=1
TRAVELATA_BASE_URL=https://api-gateway.travelata.ru
TRAVELATA_DEPARTURE_CITY_ID=2
TRAVELATA_TIMEOUT_SEC=12
TRAVELATA_MAX_RESULTS=5
```

### Основные переменные

| Переменная | Обязательна | Назначение |
| --- | --- | --- |
| `VK_TOKEN` | да | Токен сообщества ВКонтакте с доступом к сообщениям. |
| `ADMIN_IDS` | да | VK ID администраторов через запятую. |
| `MANAGER_IDS` | нет | VK ID менеджеров через запятую. |
| `DB_PATH` | нет | Путь к SQLite-базе данных. |
| `RATE_LIMIT_COUNT` | нет | Количество сообщений за период антифлуда. |
| `RATE_LIMIT_WINDOW_SEC` | нет | Окно антифлуда в секундах. |
| `EXTERNAL_TOURS_ENABLED` | нет | Включение внешнего поиска туров. |

Минимально для запуска достаточно заполнить `VK_TOKEN` и `ADMIN_IDS`. Если нужно выдать доступ менеджерам, их VK ID указываются в `MANAGER_IDS` через запятую. Администратор автоматически получает права менеджера.


## Роли

### Клиент

Клиент работает с главным меню и может:

- подобрать тур;
- открыть каталог туров;
- оставить заявку;
- посмотреть статус заявки;
- сохранить туры в избранное;
- сравнить избранные туры;
- открыть FAQ, контакты, оплату и бронирование;
- связаться с менеджером.

### Менеджер

Менеджер открывает рабочую панель командой:

```text
/staff
```

Основные возможности менеджера:

- просмотр заявок;
- открытие карточки заявки;
- взятие следующей заявки;
- закрепление заявки;
- ответ клиенту;
- отправка шаблонного ответа;
- закрытие обработанной заявки;
- просмотр загрузки менеджеров.

### Администратор

Администратор открывает панель командой:

```text
/admin
```

Администратор может управлять менеджерами, пользователями, турами, текстовыми блоками, выгрузками, PDF КП, рассылками и разблокировкой пользователей.

## Очистка

Удалить временные файлы, логи, lock-файл и экспорты:

```powershell
.\cleanup_bot.bat
```

Полная очистка с удалением базы данных:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\cleanup.ps1 -ResetDatabase
```

## Docker

Сборка образа:

```powershell
docker build -t viare-travel-vk-bot .
```

Запуск контейнера с переменными окружения:

```powershell
docker run --env-file .env viare-travel-vk-bot
```

## Примечание

Рабочие данные проекта создаются в папке `data/`. Файл `.env`, локальная база данных, логи, экспорты, виртуальное окружение и кэш не предназначены для публикации в репозитории.
